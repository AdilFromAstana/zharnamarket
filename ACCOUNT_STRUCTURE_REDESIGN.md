# ACCOUNT_STRUCTURE_REDESIGN

> План упрощения структуры аккаунта: убираем лишнюю сущность «профиль юзера», оставляем симметричную модель «публичные сущности (креатор/бизнес) + приватный кабинет». Аналог hh.ru: резюме + вакансии компании + личный кабинет.

---

## 1. Принцип

Один юзер = одна учётка, два публичных «лица» + один приватный кабинет.

- **Как креатор** → `CreatorProfile` (одно или несколько «резюме»)
- **Как бизнес** → `Ad` (одно или несколько «заданий»)
- **Как владелец аккаунта** → `/cabinet` (только я вижу)

Отдельной «страницы юзера» нет. Нет дублирования между `/profile/[id]` и `/creators/[id]`.

---

## 2. Итоговая карта URL

### Публичное (видят все)
- `/ads` — каталог заданий
- `/ads/[id]` — страница задания + блок «другие задания заказчика»
- `/creators` — каталог креаторов
- `/creators/[id]` — страница креатор-профиля + блок «другие профили автора»

### Приватное (только я, за авторизацией)
- `/cabinet` — дашборд обеих ролей
- `/cabinet/balance` — кошелёк
- `/cabinet/settings` — настройки аккаунта

### Создание/редактирование (только владелец)
- `/ads/new`, `/ads/[id]/edit`
- `/creators/new`, `/creators/edit?id=[id]`

### Удалить
- `/profile/[id]` → редирект на `/creators/[id]` (если есть опубликованный креатор-профиль) или на `/ads` с фильтром по автору

---

## 3. Что видят другие люди vs я сам

### Другие люди (публично)

**`/creators/[creatorProfileId]`** — «резюме»
- Био, аватар, категории, платформы, цены, примеры работ, отзывы
- Блок «Об авторе»: имя, рейтинг, «с такого-то года»
- Блок «Ещё профили этого автора» — карточки других его `CreatorProfile`
- Кнопка «Связаться» / «Пригласить на задание»

**`/ads/[adId]`** — «вакансия»
- Описание задания, бюджет, категория, дедлайн, платформа
- Блок «О заказчике»: аватар, имя, рейтинг, количество заданий
- Блок «Другие задания заказчика» — карточки других его `Ad`
- Кнопка «Откликнуться»

**Важно:** роли одного юзера публично НЕ связаны. На `/creators/[id]` не видно, что юзер размещает задания, и наоборот. Приватность ролей по умолчанию.

### Я сам (приватно)

**`/cabinet`** — главный приватный экран. Две симметричные секции:

*Секция «Я — бизнес»* (если есть `AdvertiserProfile`)
- Мои задания: активные / черновики / архив
- Отклики на мои задания
- CTA «+ Создать задание»

*Секция «Я — креатор»* (если есть `CreatorProfile[]`)
- Мои креатор-профили
- Отправленные отклики
- Текущие сделки, что сдать
- CTA «+ Создать креатор-профиль»

*Сверху:* баланс, быстрые действия, уведомления.

Если роль не используется — её секция свёрнута в один CTA «Разместить задание» / «Стать креатором».

**`/cabinet/balance`** — история транзакций, пополнение, вывод.

**`/cabinet/settings`** — имя, email, телефон, пароль, уведомления.

## 4. Почему это удобно (обоснование)

1. **Снимает когнитивную нагрузку.** Сейчас: «профиль», «кабинет», «креатор» — три слова, юзер гадает. Станет: «моё резюме / моё задание» (публичное) + «мой кабинет» (приватное). Дихотомия как на hh.
2. **Симметрия двух ролей.** Одинаковая логика: создать → опубликовать → редактировать из кабинета → другие видят публичную страницу. Переключаться легко.
3. **Нет лишних сущностей.** Агрегат «все публичные штуки юзера» не нужен отдельной страницей — он встроен в блоки «другие задания / другие профили автора» прямо на `/ads/[id]` и `/creators/[id]`. Контекст не теряется.
4. **Приватное под `/cabinet/*`.** Всё «моё» в одном URL-префиксе. Юзер не путается, публичное это или нет.
5. **Приватность ролей.** Две роли одного юзера не связаны публично, пока он сам не захочет. Бизнес-заказчик может не хотеть, чтобы подрядчики видели его креатор-профиль.
6. **Меньше кода.** Один route и шаблон меньше (`/profile/[id]` удаляется, `/u/[id]` не создаётся).

## 5. Пошаговый план миграции

### Шаг 1. Удалить `/profile/[id]`
- Файлы: `app/profile/[id]/page.tsx` и всё под `app/profile/`
- В `next.config.js` / middleware добавить редирект:
  - если у юзера есть опубликованный `CreatorProfile` → `/creators/[firstPublishedId]`
  - иначе → `/ads?author=[userId]` (фильтр по автору заданий)
  - если нет ни того ни другого → `/` с флеш-сообщением «Этот пользователь пока ничего не опубликовал»

### Шаг 2. Добавить блок «другие профили автора» на `/creators/[id]`
- Файл: `app/creators/[id]/page.tsx`
- Запрос: все `CreatorProfile` где `userId === currentCreator.userId` и `status === 'published'`, кроме текущего
- Если пусто — блок не показываем
- Компонент: переиспользовать существующую карточку креатора из каталога

### Шаг 3. Добавить блок «другие задания заказчика» на `/ads/[id]`
- Файл: `app/ads/[id]/page.tsx` и/или `app/ads/[id]/AdSidebar.tsx`
- Запрос: все `Ad` где `advertiserProfileId === currentAd.advertiserProfileId` и `status === 'active'`, кроме текущего
- Если пусто — блок не показываем
- Компонент: переиспользовать карточку из `/ads`

### Шаг 4. Перенести `/balance` под `/cabinet/balance`
- Переместить `app/balance/*` → `app/cabinet/balance/*`
- Редирект `/balance` → `/cabinet/balance`
- Обновить все внутренние ссылки (grep по `/balance`)

### Шаг 5. Перенести `/settings` под `/cabinet/settings`
- Переместить `app/settings/*` → `app/cabinet/settings/*`
- Редирект `/settings` → `/cabinet/settings`
- Обновить все внутренние ссылки (grep по `/settings`)

### Шаг 6. Почистить хедер
- Файл: `components/layout/AppHeader.tsx`
- Убрать пункт «Мой профиль» из меню юзера
- Оставить: «Мой кабинет», «Кошелёк» (ведёт на `/cabinet/balance`), «Настройки» (ведёт на `/cabinet/settings`), «Выйти»
- Мобильный drawer — симметрично

### Шаг 7. Обновить кабинет под две симметричные секции
- Файл: `app/cabinet/page.tsx`
- Секция «Я — бизнес» (показывается только если `AdvertiserProfile` существует; иначе — свёрнутый CTA «Разместить задание»)
- Секция «Я — креатор» (показывается только если `CreatorProfile[]` непустой; иначе — свёрнутый CTA «Стать креатором»)
- Сверху: баланс + быстрые действия

### Шаг 8. Тесты и проверки
- Тест: неавторизованный юзер не попадает в `/cabinet/*` (редирект на логин)
- Тест: юзер не может редактировать чужие `Ad` / `CreatorProfile` (403)
- Тест: старые ссылки `/profile/[id]`, `/balance`, `/settings` корректно редиректят
- Тест: блоки «другие сущности автора» не показываются если их нет
- Ручная проверка: пройти весь флоу юзера с одной ролью, с двумя ролями, без ролей

### Шаг 9. Очистка
- Удалить неиспользуемые компоненты профиля
- Удалить неиспользуемые API-роуты под `/api/profile/[id]` (если не нужны)
- grep по `profile` в UI-текстах — обновить копию

## 6. Edge-кейсы

- **Юзер без ролей** (только зарегистрировался): кабинет показывает два CTA «Создать задание» / «Создать креатор-профиль». Публичных страниц у него нет.
- **Юзер только-креатор**: в кабинете секция «бизнес» свёрнута. На `/creators/[id]` никто не узнаёт, что он ещё и заказчик (потому что он и не заказчик).
- **Юзер только-бизнес**: симметрично.
- **Юзер с обеими ролями**: в кабинете обе секции раскрыты, но на публичных страницах роли не связаны.
- **Юзер удалил `CreatorProfile`**: старые ссылки на `/creators/[deletedId]` → 404 или «профиль снят». Блоки «другие профили автора» у других креаторов этого юзера обновляются автоматически.

---

## 7. Что НЕ делаем (отложено)

- **Страница `/u/[id]` (агрегат всех публичных сущностей юзера).** Не нужна, пока нет реального кейса «юзер хочет шерить одну ссылку на всё своё». Если понадобится — делаем как опциональную витрину, которую юзер сам включает в настройках.
- **Ролевой toggle в шапке (Airbnb-style «Travel / Host»).** Не нужен, пока кабинет умещает обе роли на одном экране. Если экран разрастётся — добавим toggle.
- **Переименование `AdvertiserProfile` / `CreatorProfile` в схеме.** Работает и так.
