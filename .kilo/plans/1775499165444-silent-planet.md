# Интеграция эскроу-модели в ViralAds PARTNER: Детальная спецификация

## Общая идея

Расширяем текущую платформу (доска объявлений + прямой контакт) новым режимом: **оплата за просмотры через систему**. Пользователь выбирает при создании задания — договориться самому (текущее) или платить через платформу (новое). Ничего существующего не ломаем, только дополняем.

---

## Точка ветвления: Шаг 2 создания объявления

Сейчас в `app/ads/new/page.tsx` (строка ~781, Step 2) заказчик выбирает тип бюджета:
- `fixed` — фиксированная сумма
- `per_views` — за просмотры
- `revenue` — % от продаж
- `negotiable` — по договорённости

**Изменение:** Когда выбран `per_views`, появляется дополнительный выбор:

```
┌─────────────────────────────────────────────────┐
│  Как хотите оплачивать?                         │
│                                                 │
│  ○ Договорюсь с креатором сам (текущее)         │
│    → Указываете бюджет текстом, связь напрямую  │
│                                                 │
│  ● Через платформу (гарантия оплаты)            │
│    → Деньги замораживаются на эскроу             │
│    → Креатор получает выплату автоматически      │
│    → Платформа проверяет просмотры               │
└─────────────────────────────────────────────────┘
```

Если выбрано **"Через платформу"** — активируется эскроу-режим с дополнительными полями.

**Шаг 2 (Бюджет и контакты) — ГЛАВНАЯ РАЗВИЛКА:**

```
Выбрал budgetType:
├── fixed / revenue / negotiable → текущий flow, поля не меняются
└── per_views → НОВЫЙ ВЫБОР:
    ├── "Договорюсь сам" (paymentMode=direct)
    │   → budgetDetails (текст), контакты Telegram/WhatsApp
    │   → всё как сейчас
    │
    └── "Через платформу" (paymentMode=escrow)
```

**Шаг 3 (Предпросмотр):**

| `direct` | `escrow` |
|----------|---------|
| Карточка с бюджетом текстом | Карточка с RPM, бюджетом, калькулятором |
| "Опубликовано на 7 дней" | "Активно до исчерпания бюджета или дедлайна" |
| Контакты видны | Контакты скрыты (связь через подачу) |

**Шаг 4 (Оплата) — РАЗВИЛКА СУММЫ:**

| `direct` | `escrow` |
|----------|---------|
| Сумма: **990 ₸** (публикация 7 дней) | Сумма: **{totalBudget} ₸** (заморозка на эскроу) |
| Текст: "Публикация на 7 дней" | Текст: "Бюджет замораживается. Выплаты креаторам — по факту просмотров." |
| PaymentType: `ad_publication` | PaymentType: `escrow_deposit` |
| После оплаты: `expiresAt = +7 дней` | После оплаты: задание живёт до дедлайна или исчерпания бюджета |

### Развилка 5: Модуль оплаты (платежи)

**Файлы:** `lib/payment-client.ts`, `app/api/payments/webhook/route.ts`

**Сейчас:** 3 типа платежей — `ad_publication` (990₸), `creator_publication` (990₸), `ad_boost` (1990-3990₸).

**Добавляются:**

| PaymentType | Сумма | Что происходит после webhook |
|-------------|-------|------------------------------|
| `escrow_deposit` | `totalBudget` (5000-1000000₸) | Создать EscrowAccount, задание → `active` |
| `escrow_topup` | Произвольная | Увеличить `initialAmount` и `available` в EscrowAccount |

**Webhook обработка (`app/api/payments/webhook/route.ts`):**

```
Получен webhook:
├── ad_publication → текущий flow (active, expiresAt +7 дней)
├── creator_publication → текущий flow (isPublished: true)
├── ad_boost → текущий flow (создать AdBoost)
├── escrow_deposit (NEW):
│   → Создать EscrowAccount { initialAmount, available, status: active }
│   → Ad.status = active (НЕТ expiresAt — живёт до дедлайна/исчерпания)
│   → Ad.paymentMode = escrow
└── escrow_topup (NEW):
    → EscrowAccount.initialAmount += amount
    → EscrowAccount.available += amount
    → Если Ad.status === budget_exhausted → status = active
```

## 5. Ключевая бизнес-логика

### 5.1 Расчёт выплаты (при одобрении модератором)

```typescript
// В app/api/admin/submissions/[id]/approve/route.ts

const grossAmount = (approvedViews / 1000) * ad.rpm
const cappedGross = Math.min(grossAmount, escrow.available) // не больше остатка
const commission = cappedGross * PLATFORM_COMMISSION_RATE    // 20%
const payout = cappedGross - commission

// Атомарная транзакция:
await prisma.$transaction(async (tx) => {
  // 1. Обновить эскроу
  const escrow = await tx.escrowAccount.findUnique({
    where: { adId }, select: { available: true }
  })
  // SELECT FOR UPDATE эквивалент через transaction isolation
  
  // 2. Списать с эскроу
  await tx.escrowAccount.update({
    where: { adId },
    data: {
      spentAmount: { increment: cappedGross },
      reservedAmount: { decrement: submission.reservedAmount },
      available: { decrement: cappedGross - submission.reservedAmount }
    }
  })
  
  // 3. Зачислить на баланс креатора
  await tx.creatorBalance.upsert({
    where: { userId: submission.creatorId },
    create: { userId: submission.creatorId, balance: payout, totalEarned: payout },
    update: { balance: { increment: payout }, totalEarned: { increment: payout } }
  })
  
  // 4. Записать транзакцию
  await tx.balanceTransaction.create({
    data: {
      balanceId, type: 'earning', amount: payout,
      description: `Задание "${ad.title}" — ${approvedViews} просмотров`,
      submissionId: submission.id
    }
  })
  
  // 5. Обновить подачу
  await tx.videoSubmission.update({
    where: { id: submission.id },
    data: {
      status: 'approved', approvedViews, payoutAmount: payout,
      commissionAmount: commission, grossAmount: cappedGross,
      moderatorId, moderatedAt: new Date()
    }
  })
  
  // 6. Проверить: если available = 0 → budget_exhausted
  if (escrow.available - cappedGross <= 0) {
    await tx.ad.update({ where: { id: adId }, data: { status: 'budget_exhausted' } })
    await tx.escrowAccount.update({ where: { adId }, data: { status: 'exhausted' } })
  }
})
```

### 5.2 Резервирование при подаче

```typescript
// В app/api/tasks/[id]/submissions/route.ts

const expectedPayout = (claimedViews / 1000) * ad.rpm
const cappedPayout = ad.maxViewsPerCreator
  ? Math.min(expectedPayout, (ad.maxViewsPerCreator / 1000) * ad.rpm)
  : expectedPayout

// Атомарно зарезервировать
await prisma.$transaction(async (tx) => {
  const escrow = await tx.escrowAccount.findUnique({ where: { adId } })
  
  if (escrow.available < cappedPayout) {
    // Частичное резервирование
    reservedAmount = escrow.available
  } else {
    reservedAmount = cappedPayout
  }
  
  await tx.escrowAccount.update({
    where: { adId },
    data: {
      reservedAmount: { increment: reservedAmount },
      available: { decrement: reservedAmount }
    }
  })
  
  await tx.videoSubmission.create({
    data: { ..., reservedAmount, slaDeadline: addHours(now, 24) }
  })
})
```
