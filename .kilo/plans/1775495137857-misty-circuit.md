# Система жалоб ("Пожаловаться") — Анализ и план расширения

## Текущее состояние

### Prisma-модель `Report`

```prisma
enum ReportTargetType {
  ad
  creator
  customer
}

model Report {
  id           String           @id @default(cuid())
  submitterId  String           // FK на User — кто подал жалобу
  targetType   ReportTargetType // ad | creator | customer
  targetId     String           // ID объекта, на которого жалоба
  reason       String           // "spam: описание" или просто "spam"
  resolved     Boolean          @default(false)
  createdAt    DateTime         @default(now())
}
```

**Поля:**
| Поле | Тип | Описание |
|------|-----|----------|
| `id` | `cuid` | Первичный ключ |
| `submitterId` | `String` | Кто подал жалобу (FK → User) |
| `targetType` | `ReportTargetType` | Тип объекта: `ad`, `creator`, `customer` |
| `targetId` | `String` | ID объекта (объявления, креатора, заказчика) |
| `reason` | `String` | Причина, формат `"reasonKey: description"` |
| `resolved` | `Boolean` | Обработана ли (по умолчанию `false`) |
| `createdAt` | `DateTime` | Дата создания |

### Допустимые причины жалоб

Определены в `app/api/reports/route.ts`:
- `spam` — Спам
- `inappropriate` — Неуместное
- `fake` — Фейк
- `scam` — Мошенничество
- `harassment` — Харассмент
- `other` — Другое

### API-эндпоинты

| Метод | Путь | Файл | Описание |
|-------|------|------|----------|
| `POST` | `/api/reports` | `app/api/reports/route.ts` | Подать жалобу (авторизация + rate limit 5/15мин) |
| `GET` | `/api/admin/reports` | `app/api/admin/reports/route.ts` | Список жалоб для админа (фильтры, пагинация) |
| `PATCH` | `/api/admin/reports/[id]` | `app/api/admin/reports/[id]/route.ts` | Пометить жалобу как обработанную/необработанную |

### Админ-панель

Полноценная страница `/admin/reports` (`app/admin/reports/page.tsx`):
- Фильтры по статусу (обработанные/необработанные) и типу (объявления/креаторы/заказчики)
- Карточки жалоб с информацией о подавшем, причине, описании
- Кнопка "Обработано" / "Открыть заново"
- Ссылка "Перейти" к объекту жалобы
- Пагинация, бейдж с количеством необработанных

## Проблемы

1. **Кнопки "Пожаловаться" — фейковые.** Обе кнопки (на объявлении и креаторе) просто показывают тост, НЕ вызывая `POST /api/reports`.
2. **Нет UI-формы для жалобы.** Нет модалки/формы, где пользователь выбирает причину и может добавить описание.
3. **Нет кнопки на странице заказчика.** `targetType: "customer"` поддерживается в API, но на UI кнопки нет.
4. **Отзывы нельзя зарепортить.** `review` нет в `ReportTargetType`, нет кнопки в `ReviewCard`.
5. **Нет тестов для `POST /api/reports`.**
6. **Нет связи между жалобами и блокировкой.** Поля `blocked`/`blockedAt` в User не связаны с системой жалоб.

## План реализации

### Шаг 1: Создать универсальный компонент `ReportModal`

**Файл:** `components/report/ReportModal.tsx`

Модальное окно с:
- Выбором причины (radio buttons из `VALID_REASONS`)
- Полем описания (textarea, макс 2000 символов)
- Кнопками "Отправить" / "Отмена"
- Состояниями загрузки и ошибок

**Props:**
```ts
interface ReportModalProps {
  open: boolean;
  onClose: () => void;
  targetType: "ad" | "creator" | "customer" | "review"; // расширяемый
  targetId: string;
}
```

### Шаг 2: Расширить `ReportTargetType` на `review`

**Файл:** `prisma/schema.prisma`
```prisma
enum ReportTargetType {
  ad
  creator
  customer
  review    // NEW
}
```

Обновить `VALID_TARGET_TYPES` в `app/api/reports/route.ts`.
Обновить `TARGET_TYPE_LABELS` в `app/admin/reports/page.tsx`.

### Шаг 3: Подключить `ReportModal` к существующим кнопкам

| Место | Файл | Действие |
|-------|------|----------|
| Объявление | `app/ads/[id]/AdDetailClient.tsx` | Заменить `toast.info` на открытие `ReportModal` с `targetType="ad"` |
| Креатор | `app/creators/[id]/CreatorDetailClient.tsx` | Заменить `toast.info` на открытие `ReportModal` с `targetType="creator"` |

### Шаг 4: Добавить кнопку "Пожаловаться" на новые страницы

| Место | Файл | `targetType` |
|-------|------|-------------|
| Профиль заказчика | `app/customers/[id]/...` (нужно найти точный файл) | `customer` |
| Карточка отзыва | `components/ReviewCard.tsx` или аналог | `review` |

### Шаг 5: Обновить админ-панель жалоб

- Добавить лейбл для `review` в `TARGET_TYPE_LABELS`
- Добавить фильтр "Отзывы" в фильтры по типу
- Добавить ссылку "Перейти" для отзывов (навигация к отзыву)

### Шаг 6: Миграция БД

```bash
npx prisma migrate dev --name add-review-to-report-target-type
```

## Какие модели должны поддерживать жалобы?

| Модель | Поддержка | Обоснование |
|--------|-----------|-------------|
| **Ad** (объявление) | Да (уже есть в enum) | Может содержать мошенничество, спам |
| **CreatorProfile** | Да (уже есть в enum) | Фейковый профиль, мошенничество |
| **AdvertiserProfile** (заказчик) | Да (уже есть в enum) | Фейковый заказчик, мошенничество |
| **Review** (отзыв) | **Добавить** | Фейковые отзывы, оскорбления |
| PaymentSession | Нет | Внутренняя сущность |
| PromoCode | Нет | Админская сущность |
| Session | Нет | Техническая сущность |
