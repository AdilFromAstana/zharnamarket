# Fix for Category and City Validation Mismatch

## Problem
The frontend sends Russian language strings for cities and categories (e.g., "Астана", "Мемы") but the backend validation expects English enum values (e.g., "Astana", "Memy"). This causes validation errors when creating/editing creator profiles.

## Root Cause
- Frontend constants (`lib/constants.ts`) use Russian strings for UI display
- Backend validation (`lib/validation.ts`) uses English enum values matching Prisma schema
- Form components use the same string for both `label` and `value`, submitting Russian strings to backend

## Solution
Create mappings between Russian UI strings and English enum values, then:
1. Use Russian strings for `label` (display)
2. Use English strings for `value` (form submission)
3. Update sidebar and other components to convert English values back to Russian for display

## Changes Required
