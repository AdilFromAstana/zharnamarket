# План: Уменьшить gap в grid категорий на мобильном

## Проблема
`grid grid-cols-1 md:grid-cols-3 gap-4` — на мобильном все 3 Select стекаются вертикально, и `gap-4` (16px) между ними выглядит как слишком большое пустое пространство.

## Файлы (2 места)
- `app/ads/new/page.tsx` строка 1121
- `app/ads/[id]/edit/page.tsx` строка 327

## Фикс
Заменить `gap-4` → `gap-0 md:gap-4` в обоих файлах.

- Мобильный: `gap-0` = 0px (у Ant Design Form.Item есть свой `margin-bottom`)
- Десктоп: `gap-4` = 16px (без изменений)

# План: Фиксация фильтров — videoFormat / adFormat / adSubject в AdFilters

## Корневые причины (3 файла)

### 1. `app/ads/page.tsx` — SSR запрос не считает новые категории
- `videoFormat: {}`, `adFormat: {}`, `adSubject: {}` — жёстко прописаны пустыми
- URL-параметры `?videoFormat=...&adFormat=...&adSubject=...` — никогда не парсятся
- `fetchAdsPage()` не получает эти параметры → не фильтрует, не считает facets

**Что добавить:**
```ts
// В AdsPage:
const videoFormats = parseCSV(params.videoFormat);
const adFormats    = parseCSV(params.adFormat);
const adSubjects   = parseCSV(params.adSubject);

// В fetchAdsPage — параметры:
videoFormats, adFormats, adSubjects

// В where:
...(videoFormats.length && { videoFormatId: { in: videoFormats } }),
...(adFormats.length    && { adFormatId:    { in: adFormats } }),
...(adSubjects.length   && { adSubjectId:   { in: adSubjects } }),

// Facet wheres — каждый исключает своё измерение (аналогично platform/city)
// 3 новых groupBy в Promise.all:
prisma.ad.groupBy({ by: ["videoFormatId"], where: vfFacetWhere, _count: true }),
prisma.ad.groupBy({ by: ["adFormatId"],    where: afFacetWhere, _count: true }),
prisma.ad.groupBy({ by: ["adSubjectId"],   where: asFacetWhere, _count: true }),

// В facets:
videoFormat: Object.fromEntries(vfCounts.map(r => [r.videoFormatId!, r._count])),
adFormat:    Object.fromEntries(afCounts.map(r => [r.adFormatId!,    r._count])),
adSubject:   Object.fromEntries(asCounts.map(r => [r.adSubjectId!,   r._count])),
```

### 2. `app/ads/AdsListClient.tsx` — новые параметры не попадают в URL и в API-запрос

**`buildURL`** — добавить:
```ts
if (filters.videoFormat?.length) params.set("videoFormat", filters.videoFormat.join(","));
if (filters.adFormat?.length)    params.set("adFormat",    filters.adFormat.join(","));
```

### 3. `AdFilters.tsx` — 3 новые секции скрыты, старая категория устарела

**Удалить:** секцию "Категория контента" (использует deprecated enum значения)

**Переработать `DynamicFacetSection`:**
- Было: показывает только элементы из `facetCounts` (если facets пустой — секция не рендерится)
- Стало: загружает ВСЕ активные элементы из API, показывает counts из facets (0 если нет), отключает нулевые — точно как `FilterSection`

```tsx
function DynamicFacetSection({ title, endpoint, facetCounts, selected, onToggle }) {
  const [items, setItems] = useState<FilterItem[]>([]);
  
  useEffect(() => {
    fetch(endpoint)
      .then(r => r.json())
      .then(res => setItems((res.data ?? []).map(i => ({ label: i.label, value: i.id }))));
  }, [endpoint]);

  return (
    <FilterSection
      title={title}
      items={items}
      selected={selected}
      onToggle={onToggle}
      counts={facetCounts}    // ID → count маппинг
    />
  );
}
```

**Убрать** условные рендеры `Object.keys(facets.videoFormat).length > 0` — секции теперь всегда показываются (если в API есть активные записи).

# План: Вертикальный список "Похожих объявлений" на мобильном

## Почему вертикальный список, а не карусель (NN/G research)

Горизонтальная карусель плохо работает когда:
- Элементов больше 4 (у нас до 6)
- Нет изображений — текстовые карточки плохо читаются в узкой колонке
- Пользователь уже устал скроллить вниз до этой секции

**Вердикт NN/G (2018):** *"If you have a high number of items, use a list view instead and allow people to directly access any of the items on the page."*

Паттерн: компактные строки списка, как "похожие вакансии" на Indeed / HH.ru / LinkedIn.

# (Архив) План: Система категорий — 3 измерения + Admin CRUD

## Архитектурное решение

**Текущий подход:** категории — Prisma enum (хардкод). Нельзя добавлять/редактировать без деплоя.

**Новый подход:** категории — **таблицы в БД**. Админ управляет через CRUD-панель. Enum-подход остаётся для обратной совместимости (миграция данных).
