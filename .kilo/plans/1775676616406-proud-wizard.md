# Редизайн страницы Кабинета (Cabinet Dashboard)

## UX/UI Анализ текущей страницы

### Проблемы с точки зрения пользователя

**1. Нет визуальной иерархии** — все блоки выглядят одинаково по приоритету:
- Приветствие, два карточных блока, кошелёк, быстрые действия — всё на одном уровне
- Самая важная информация (баланс) спрятана почти в самом низу
- Нет "сканируемости" — глаз не знает куда смотреть первым

**2. Путаница ролей** — платформа обслуживает 2 типа пользователей:
- **Бизнес** (создаёт задания, ищет креаторов) 
- **Креатор** (создаёт профили, откликается на задания)
- Оба блока показаны с равным весом — пользователь не понимает свой primary path
- Пустые состояния для обеих ролей одновременно выглядят как "тут ничего нет"

**3. Несогласованность с другими страницами сайта:**
- Лендинг (`/`): hero → stats → контент → CTA-баннер (чёткий поток)
- Управление заданиями (`/ads/manage`): заголовок + CTA → stats grid → табы + контент
- Баланс (`/balance`): gradient-карточка → статистика → список операций
- Кабинет (`/cabinet`): приветствие → два карда → платежи → отзывы → кошелёк → быстрые действия (плоская структура без логики)

**4. Дублирование** — быстрые действия внизу дублируют навигацию из хедера:
- "Создать задание" есть и в карточке, и в быстрых действиях, и в хедере
- "Мои профили" — то же самое

**5. Неэффективное использование пространства:**
- Приветствие занимает отдельную строку без дополнительной пользы
- Кошелёк — маленький блок, но стоит отдельно
- Для нового пользователя 70% экрана — пустые состояния

## План изменений

### Секция 1: Hero + Кошелёк (объединить)

**Текущее:** Приветствие отдельно сверху, кошелёк отдельно внизу.

**Новое:** Одна строка — приветствие слева, баланс кошелька справа. На мобиле — стек.

```
┌──────────────────────────────────────────────────────┐
│  😊 Привет, Test!              │  💰 0 ₸            │
│  creator@test.local            │  [Пополнить] [Операции] │
└──────────────────────────────────────────────────────┘
```

**Стиль:** `bg-gradient-to-br from-sky-50 via-white to-blue-50 rounded-2xl border border-sky-100 p-6` — мягкий градиент как акцентный блок (согласуется с balance page и quick actions).

**Мобиль:** `flex-col`, баланс под приветствием с разделителем.

---

### Секция 2: Статистика (новый блок)

**Текущее:** Нет сводной статистики.

**Новое:** Сетка 2x2 (мобиль) / 4 колонки (десктоп) — как на `/ads/manage`.

```
┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐
│  0       │ │  0       │ │  0       │ │  0       │
│ Заданий  │ │ Профилей │ │ Просмотров│ │ Обращений│
└──────────┘ └──────────┘ └──────────┘ └──────────┘
```

**Данные:** Активные задания, опубликованные профили, суммарные просмотры, суммарные обращения/отклики. Всё доступно из уже загруженных данных (myAds, myProfiles).

**Стиль:** Каждая карточка имеет **цветной фон** (свой цвет) и **белый текст** (и число, и label, и иконка).

Конкретно менять в `app/cabinet/page.tsx` (Секция 2: Статистика):
- Убрать `bg-white border border-gray-200` с карточек
- Добавить каждой карточке свой `bg` + `border` в тон:
  - Активных заданий: `bg-green-500 border-green-600`
  - Профилей: `bg-violet-500 border-violet-600`
  - Просмотров: `bg-sky-500 border-sky-600`
  - Обращений: `bg-amber-500 border-amber-600`
- Label (`text-xs text-gray-400`) → `text-white/80`
- Value (`style={{ color: stat.color }}`) → `text-white`
- Icon (цветные классы `text-green-500` и т.д.) → `text-white/80`

---

### Секция 3: Мои задания + Мои профили (улучшенные)

**Фикс: иконки в empty state слишком бледные — сделать чёрными (text-gray-900)**

В `app/cabinet/page.tsx`:
- Строка 301: `<FileTextOutlined className="text-sky-400 text-2xl" />` → `text-gray-900 text-2xl`
- Строка 400: `<UserOutlined className="text-purple-400 text-2xl" />` → `text-gray-900 text-2xl`

**Текущее:** Два карда 50/50 с минималистичными пустыми состояниями.

**Новое:** Те же два карда, но с улучшениями:
- Добавить микро-статистику в заголовок (количество через Tag)
- Улучшить пустые состояния: добавить крупную иконку, более мотивирующий текст
- Для непустых состояний: показывать 2 item + "Все задания →" / "Все профили →"
- **Основная структура остаётся** — она хорошо работает, просто нуждается в полировке

**Стиль:** Без изменений — `bg-white rounded-2xl border border-gray-200 p-6`.

---

### Секция 4: Последние операции (объединить платежи + кошелёк)

**Текущее:** История платежей — отдельная секция, кошелёк — отдельная секция.

**Новое:** Убрать отдельный кошелёк (он перенесён в hero). Оставить историю платежей с ограничением 3 записи (вместо 5) и ссылкой "Все операции →" на `/balance`.

**Стиль:** Без изменений, просто перемещение.

---

### Секция 5: Отзывы (без изменений)

Условный блок, показывается только при наличии отзывов. Оставить как есть.

---

### Секция 6: Быстрые действия (редизайн)

**Текущее:** 4 одинаковые карточки-квадраты с иконками.

**Новое:** Сделать действия контекстными и убрать дублирование:
- Если нет заданий → акцент на "Создать задание"
- Если нет профилей → акцент на "Создать профиль"  
- Если есть и то и другое → показать "Найти креатора" и "Смотреть задания"
- Убрать "Мои профили" — это дублирует карточку выше

**Стиль:** Оставить `bg-gradient-to-r from-sky-50 to-blue-50 rounded-2xl`.

