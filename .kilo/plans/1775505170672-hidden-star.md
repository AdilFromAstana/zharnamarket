# План: Escrow-задание — бюджет + вкладка "Видео"

## Цель
Для заданий с `paymentMode === "escrow"` на странице `/ads/[id]`:
1. **Секция бюджета** — прогресс-бар + статистика эскроу (всегда видна, над вкладками)
2. **Вкладка "Видео"** — список одобренных подач с превью, ссылкой и суммой выплаты

---

## Анализ текущего состояния

### Что уже есть:
- `GET /api/tasks/[id]/submissions` — возвращает все подачи, но только owner/admin
- `VideoSubmission` содержит: `videoUrl`, `screenshotUrl`, `approvedViews`, `grossAmount`, `payoutAmount`, `status`
- `AdDetailClient.tsx` — нет вкладок, один layout для всех типов заданий
- `page.tsx` — SSR, загружает объявление и похожие через Prisma напрямую

### Чего не хватает:
- Публичной загрузки одобренных подач (только для `approved` статуса)
- Системы вкладок в `AdDetailClient.tsx` для escrow-заданий
- Компонента карточки видео
- Лайки/комментарии в системе **не хранятся** — показываем только просмотры

---

## Архитектурные решения

### Загрузка данных: Server-side в `page.tsx`
Преимущества SSR:
- Нет дополнительного круга запросов
- SEO — список видео в HTML
- Пользователь видит актуальные данные при каждом обновлении страницы (это и есть «каждый раз как обновляет»)

Загружаем только `approved` подачи через Prisma в `page.tsx`, передаём в `AdDetailClient` как prop `approvedSubmissions`.

### Секция бюджета (EscrowBudgetSection)
Всегда видна для escrow-заданий — **над вкладками**, в верхней части левой колонки (вместо текущего тёмного budget-блока). Состоит из:

**Строка 1 — прогресс:**
```
"12 080 ₸ из 100 000 ₸  Выплачено креаторам  12%"
[████████░░░░░░░░░░░░░░░░░░░░░] progress bar
● Выплачено  ● В резерве  │ Лимит выплат
```
- Зелёный сегмент = `escrowAccount.spentAmount / initialAmount`
- Серый сегмент = `escrowAccount.reservedAmount / initialAmount` (в модерации)
- Процент = `(spentAmount + reservedAmount) / initialAmount * 100`

**Строка 2 — бейджи:**
```
[Instagram] [50 ₸ за 1 000 👁] [до 15 апр]
```
- Платформа (badge)
- RPM бейдж: `{rpm} ₸ за 1 000 просмотров`
- Дедлайн (если `submissionDeadline` задан)

**Строка 3 — 3 карточки-метрики:**
```
┌──────────┐  ┌──────────┐  ┌──────────────────┐
│  📹 2    │  │ 👁 302K  │  │ 💰 84 800 ₸       │
│  Видео   │  │ Просм.   │  │ Осталось из 100K  │
└──────────┘  └──────────┘  └──────────────────┘
```
- `approvedSubmissions.length` — кол-во одобренных видео
- `totalApprovedViews` — сумма `approvedViews` по всем approved submissions (считается в `page.tsx`)
- `escrowAccount.available` — остаток бюджета

### Структура вкладок
Только для `paymentMode === "escrow"`. Вкладки:
- **"Задача"** — текущий контент (описание, контакты, детали)
- **"Видео (N)"** — список одобренных подач

**Важно:** текущий тёмный budget-блок (`bg-slate-900`) убирается для escrow-заданий — его роль берёт на себя `EscrowBudgetSection`.

Состояние вкладки: `useState` на клиенте.

## Дизайн карточки видео

### Данные для отображения:
| Поле | Источник |

#### Цвета для badge тематики и типа бюджета

**Проблема с добавлением цветов:**
Boost-карточки (`premium`, `highlight`, `rise`) уже инжектируют amber/violet/sky цвета в бейджи тематики и типа бюджета:
```
boostLevel === "premium" → bg-amber-50 border-amber-200 text-amber-600
boostLevel === "highlight" → bg-violet-50 border-violet-200 text-violet-500
boostLevel === "rise" → bg-sky-50 border-sky-200 text-sky-500
```
Если добавить цвета **по значению** (напр. Кино-нарезки = синий, Красота = розовый), они вступят в конфликт с цветами буст-уровня. Карточка с буст-уровнем `premium` (amber) и категорией с синим цветом будет выглядеть рассогласованно.

**Три варианта:**

**Вариант A — Не трогать (рекомендуется)**
Тематика и тип бюджета остаются серыми (нейтральными). Платформа и тип оплаты уже цветные — этого достаточно. Профессионально, без визуального шума. Так работают Vercel, Linear, Notion.
```
[Кино-нарезки]gray  [За просмотры]gray  [Instagram]gradient  [⚡ Через платформу]emerald
```

**Вариант B — Цвет по измерению (dimension color)**
Каждый тип бейджа получает свой фиксированный цвет — НЕ по значению, а по типу (все категории = один цвет, все типы бюджета = другой цвет). Применять только для **небустованных** карточек, для буст-карточек оставить текущий amber/violet/sky.
```
Тематика → bg-indigo-50 border-indigo-200 text-indigo-600
Тип бюджета → bg-amber-50 border-amber-200 text-amber-600
```
Риск: amber для типа бюджета совпадает с цветом premium-буста — размытие смысла.

**Вариант C — Цвет по значению**
Каждая категория/тип бюджета = уникальный цвет. Полная радуга. Визуально активно, но:
- 15+ категорий × 4 типа бюджета = много цветов
- Конфликт с boost-цветами
- Не рекомендуется

**Итог:** Вариант A — самый чистый. Если хочется чуть больше жизни — Вариант B только для тематики (indigo), тип бюджета оставить серым (нет путаницы с premium-amber).
