# Fix: Sticky Step Header in /ads/new

## Problem

The sticky step header at `app/ads/new/page.tsx:918` has two CSS bugs:

### 1. `top-0` overlaps the AppHeader

`AppHeader` (`components/layout/AppHeader.tsx:157`) is `sticky top-0 z-50` with heights:
- Mobile: `h-14` = 56 px
- Desktop (md+): `h-16` = 64 px

The step header uses `sticky top-0 z-10`, so when scrolled it tries to pin at the very top of the viewport ÔÇö the same position as AppHeader. Because `z-10 < z-50`, the step header slides *under* the AppHeader and disappears instead of sticking cleanly below it.

### 2. `my-6` on a sticky element causes a visual jump

`my-6` adds 24 px margin top/bottom. When `position: sticky` kicks in, the element jumps to `top: 0` while its flow placeholder (with margins) remains in place. The gap between the flow-placeholder and the stuck element is noticeable on scroll and looks broken.

## Fix

**Single file change ÔÇö `app/ads/new/page.tsx`, line 918**

| Class | Old | New | Reason |
|---|---|---|---|
| `top` | `top-0` | `top-14 md:top-16` | Stick below AppHeader (56 px mobile / 64 px desktop) |
| margin | `my-6` | `mb-4` | Remove margin-top from sticky element; keep a small bottom gap |

Full replacement:
```diff
- className="sticky top-0 z-10 bg-white -mx-4 px-4 sm:-mx-6 sm:px-6 lg:-mx-8 lg:px-8 pt-4 pb-4 my-6 border-b border-gray-100 shadow-sm"
+ className="sticky top-14 md:top-16 z-10 bg-white -mx-4 px-4 sm:-mx-6 sm:px-6 lg:-mx-8 lg:px-8 pt-4 pb-4 mb-4 border-b border-gray-100 shadow-sm"
```

The breadcrumb (`<Breadcrumb className="mb-6 hidden sm:block">`) above the sticky header already provides top spacing on desktop. On mobile the breadcrumb is hidden, so no top margin is needed for initial spacing ÔÇö the step header simply starts at the top of the page content.

## Additional change ÔÇö hide progress bar on mobile

The segmented progress bar (`h-1` segments + text labels) is redundant on mobile because there is already a separate dot indicator with `sm:hidden`. The progress bar should only render on `sm+`.

**File: `app/ads/new/page.tsx`, progress bar container div (inside the sticky header, `{step > 0 && ...}`)**

```diff
- <div className="flex gap-1.5 mb-3">
+ <div className="hidden sm:flex gap-1.5 mb-3">
```

`flex` Ôćĺ `hidden sm:flex` hides the bar on mobile and restores it on tablet/desktop, exactly mirroring how the dot indicator uses `sm:hidden`.
