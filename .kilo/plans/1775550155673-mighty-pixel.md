# Plan: Desktop-friendly layout for /ads/new

## Problem

`app/ads/new/page.tsx` is designed purely for mobile:
- Content is capped at `max-w-lg` (512 px) with `mx-auto`, leaving huge empty margins on desktop
- Sticky step-header uses a `-mx-4 px-4` negative-margin hack that bleeds past the layout container — looks broken on desktop
- `StickyBottomBar` is `fixed bottom-0 left-0 right-0` and spans the full screen width on desktop
- No use of the extra horizontal space available on `lg+` screens

## Target UX (desktop ≥ 1024 px)

Two-column layout inside the existing `max-w-5xl` container:
- **Left column (~60 %)**: the current form steps (wider, more breathing room)
- **Right column (~320 px)**: a contextual sidebar that changes per step (tips, payment info, order summary, trust indicators)

Navigation buttons move **inline** at the bottom of the left column on desktop; `StickyBottomBar` is hidden with `lg:hidden`.

Mobile layout is **unchanged** — everything remains a single column with the sticky bottom bar.

## Detailed Changes

### 1. `StickyBottomBar.tsx` — accept `className`

Add an optional `className?: string` prop and spread it onto the outer `<div>`.

```diff
interface StickyBottomBarProps {
  ...
+ className?: string;
}

export default function StickyBottomBar({
  ...
+ className,
}: StickyBottomBarProps) {
  ...
  return (
-   <div className="fixed bottom-0 left-0 right-0 z-50 ...">
+   <div className={`fixed bottom-0 left-0 right-0 z-50 ... ${className ?? ""}`}>
```

### 2. `app/ads/new/page.tsx` — layout restructuring

#### 2a. Fix the sticky step-header (remove mobile-only negative-margin hack)

```diff
- <div className="sticky top-0 z-10 bg-white -mx-4 px-4 pt-3 pb-3 mb-5 border-b border-gray-100 shadow-sm">
+ <div className="sticky top-0 z-10 bg-white pt-3 pb-3 mb-5 border-b border-gray-100 shadow-sm">
```

The header stays sticky within the `PublicLayout` container. On desktop this is fine and looks polished; on mobile the layout container already spans the viewport width.

Replace the current `max-w-lg mx-auto pb-28` wrapper with:

```jsx
  <div className="lg:grid lg:grid-cols-[1fr_320px] lg:gap-8">

    {/* Left column — form content (same as before, but no max-w-lg) */}
    <div className="pb-28 lg:pb-8">
      {/* step 0, form steps 1–4 ... unchanged interior */}

    {/* Right column — desktop-only contextual sidebar */}

  </div>
```

#### 2c. Add a `StepSidebar` component (inline inside the file)

A small function component that renders step-aware help content.  
Keep it inside `page.tsx` to avoid creating a new file.
