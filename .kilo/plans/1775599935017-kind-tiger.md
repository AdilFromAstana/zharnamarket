# Creator Detail Page — Mobile UI Refresh + Meta Tag Strip

## Goal
Bring `CreatorDetailClient.tsx` to the same visual quality as `AdDetailClient.tsx` on mobile:
- Colorful, semantically-coded accents on key details
- Remove redundant borders (especially on mobile where cards should be "floating" sections, not framed boxes)
- Better visual hierarchy

---

## Analysis: what AdDetailClient does well vs. CreatorDetailClient

| Pattern | AdDetailClient ✅ | CreatorDetailClient ❌ |
|---|---|---|
| Main card border on mobile | `md:bg-white md:rounded-2xl md:border md:border-gray-200 md:p-8` — borderless/bgless on mobile | `bg-white rounded-2xl border border-gray-200 p-6` — always bordered |
| Category/format tags | Color-coded pills: blue, violet, amber, emerald | Plain Ant `<Tag>` without semantic color |
| Platform links | Color-coded icon backgrounds (blue for Telegram, green for WhatsApp, etc.) | `border border-gray-100` neutral row, not colorful |
| Bio separator | No border-t, just margin | `border-t border-gray-100` inside the card |
| Pricing block | No border on budget card — dark bg defines it | `border border-green-200` — redundant with background |
| Dividers | Sparse, used only in sidebar | 2× `<Divider />` inside profile card on every screen |
| Sticky bottom bar | `bg-white/80 backdrop-blur-xl border-t border-gray-100` | `bg-white border-t border-gray-200` — heavier |
| Portfolio / section cards | Always `border border-gray-200` | Same, not responsive |

## Phase 3 — Meta tag strip (no new DB fields)

### Goal
Add a horizontal pill strip directly below the creator's name (like ad listings have) so advertisers can instantly scan platform + content type without scrolling.

### What advertisers need at a glance
Research confirms: platform (#1 filter) and content niche (#2 filter) are what brands check first. Both exist in `CreatorProfile` but are currently buried below the fold.

### Implementation

**Location:** Inside the avatar/header block, after the `<h1>` name and before the city/availability row.

```jsx
{/* Meta tag strip — platforms + content categories */}
<div className="flex flex-wrap items-center justify-center gap-1.5 mt-2.5 mb-1">
  {creator.platforms.map((p) => (
    <span
      key={p.name}
      className={`inline-flex items-center text-[10px] font-semibold uppercase tracking-wider px-2.5 py-1 rounded-full ${PLATFORM_BADGE_CLASSES[p.name] ?? "bg-gray-800 text-white"}`}
    >
      {p.name}
    </span>
  ))}
  {creator.contentCategories.map((cat) => (
    <span
      key={cat}
      className="inline-flex items-center text-[10px] font-semibold uppercase tracking-wider px-2.5 py-1 rounded-full bg-violet-50 text-violet-700"
    >
      {ENUM_TO_CATEGORY[cat] ?? cat}
    </span>
  ))}
</div>
```

**Imports to add:**
- `PLATFORM_BADGE_CLASSES` from `@/lib/constants`
- `ENUM_TO_CATEGORY` from `@/lib/enum-maps`

**Remove the bottom "Категории" section** — it's now covered by the strip above. The detailed "Платформы" section stays (it has handle + follower count, which is different detail level).

## Phase 4 — Price List redesign

### Goal
Replace the free-text label in the price list form with a structured **Platform (optional) + AdFormat (required)** selector. No schema migration — `PriceItem.label: String` stays as-is, just generated by the form.

### Why AdFormat (not VideoFormat)
- Creators price by **how the ad appears** (нативная интеграция costs more than a banner), not by what type of content they make
- VideoFormat = their content style (already shown via contentCategories + portfolio)
- AdFormat = exactly what an advertiser needs to spec and compare prices for
- 6 clean options, no confusing combinations

### Why NOT a range / "от" prefix
- `negotiable: Boolean` already exists at the profile level — covers the "prices are starting points" case
- Adding per-row complexity (min/max range) gives no real benefit and confuses creators

### Data available
- `GET /api/ad-formats` — already exists, returns `{ id, key, label, description }`
- 6 AdFormat options: Хук, Баннер на видео, Бегущая строка, Нативная интеграция, Текст на экране, Полный рекламный ролик
- `Platform` values: TikTok, Instagram, YouTube — from creator's own platforms list

---

### Form changes (create + edit)

**Replace** the free-text `<Input placeholder="TikTok-ролик до 1 мин">` in each price row with:

```
[Platform select (optional)]  [AdFormat select]  [Price ₸]  [×]
```

**Row structure:**
```jsx
// Platform — optional, narrowed to creator's own platforms
<Select placeholder="Платформа (необ.)" allowClear options={creatorPlatforms} />

// AdFormat — required, loaded from /api/ad-formats
<Select placeholder="Тип рекламы" options={adFormats} />

// Price — unchanged
<InputNumber placeholder="15000" suffix="₸" />
```

**Generated label** stored in `PriceItem.label`:  
- If platform selected: `"TikTok — Нативная интеграция"`
- If no platform: `"Нативная интеграция"`

**Quick-add preset chips** above the list (most common formats):
```
Добавить: [+ Нативная интеграция]  [+ Баннер]  [+ Хук]  [+ Полный ролик]
```
Clicking adds a row pre-filled with that AdFormat (no platform, price empty).

**Remove "Минимальная ставка" field** — it's redundant when a full price list exists.  
Keep `negotiable` toggle.

**AdFormat loading:** Simple `useEffect` + fetch inside the form component. No extra hook needed.
