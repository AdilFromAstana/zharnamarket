# План рефакторинга страницы /creators/manage

**Дата:** 2026-04-18
**Целевой файл:** [app/creators/manage/page.tsx](../app/creators/manage/page.tsx)
**Цель:** привести экран к стандартам дашбордов креатор-маркетплейсов (Fiverr, Shopify, Stripe), повысить плотность информации, убрать дубли и вывести критические действия из overflow-меню.

---

## 0. Acceptance criteria (общие)

- Все критические действия доступны не более чем за 1 клик.
- Ни одна колонка таблицы не отображается пустой в 100% строк.
- У черновика ставка не показывается как «от 0 ₸».
- Состояние фильтра таблицы синхронизировано между мобилкой и десктопом и сохраняется в URL.
- Файл страницы разбит на компоненты, ни один компонент не превышает 250 строк.
- Нет регрессий: publish, unpublish, delete, edit, boost, preview работают как раньше.
- Нет эмодзи в UI (использовать иконки из `@ant-design/icons`).

---

### Задачи

### Acceptance

- Никакой компонент не превышает 250 строк.
- Линтер и TypeScript проходят без ошибок.

## 3. Этап 3. Единый стейт фильтра + URL persistence

**Цель:** убрать рассинхрон между мобилкой и десктопом, сохранять состояние при перезагрузке.

### Задачи

1. Удалить отдельные стейты `activeFilter` и `activeTab` ([строки 60, 61](../app/creators/manage/page.tsx#L60-L61)).
2. Ввести единый `filter: FilterKey` в URL как query-параметр: `/creators/manage?filter=published`.
3. Использовать `useSearchParams` + `useRouter` из `next/navigation` для чтения и обновления.
4. Дефолт — `all`, если параметр отсутствует или невалиден.

### Acceptance

- Переключение фильтра на мобилке и десктопе меняет URL.
- Копирование URL и открытие в новой вкладке восстанавливает фильтр.
- Нет рассинхрона между обеими представлениями.

### Риски

| Риск | Митигация |
|---|---|
| `useSearchParams` требует Suspense в Next.js App Router | Обернуть компонент в Suspense boundary на уровне page.tsx. |
| Невалидный query-параметр ломает парсинг | Жёстко валидировать против `FilterKey`, фолбэк на `all`. |

## 4. Этап 4. Переделка таблицы (главный этап)

**Цель:** довести таблицу до уровня дашборда, а не справочника.

### 4.1. Новая структура колонок

| # | Колонка | Ширина | Содержимое |
|---|---|---|---|
| 1 | Профиль | auto | Аватар 32×32 + имя + verified-бейдж + title (маленьким текстом) |
| 2 | Платформы | 140 | До 3 иконок платформ с подсказкой подписчиков на hover |
| 3 | Статус | 160 | Бейдж опубликован/черновик + строка буста (если активен) |
| 4 | Ставка | 120 | Нормализованная ставка (см. этап 1) |
| 5 | Обновлён | 120 | Относительное время (`3 дня назад`) + абсолютное в title |
| 6 | Действия | 180 | Inline-иконки: Превью, Редакт, Буст. Остальное — в overflow |

### 4.2. Inline-действия

Файл: `_components/ProfileRowActions.tsx`.

- Иконки-кнопки: `EyeOutlined` (Превью), `EditOutlined` (Редакт), `RocketOutlined` (Буст, только если published и нет активного буста).
- Кнопка Буст — primary-акцент (purple), остальные — `type="text"`.
- Overflow-меню (`EllipsisOutlined`): Опубликовать/Снять с публикации, Удалить.
- Tooltip на каждую иконку с подписью действия.

### 4.3. Визуальные параметры

- `size="small"` у `<Table>` — компактная плотность.
- `pagination={{ pageSize: 20, showSizeChanger: true }}`.
- `scroll={{ x: 900 }}` — горизонтальный скролл на узких экранах.
- Skeleton loading при `loading=true` (использовать встроенный в antd Table `loading` проп).

### 4.4. Пустое состояние

- Файл: `_components/ProfileTable.tsx`.
- Компонент `<Empty>` с иллюстрацией + CTA «Создать профиль» при нулевом списке.
- Разделить пустоту «нет профилей вообще» и «нет профилей под текущий фильтр» — разные тексты.

### Acceptance

- Все критические действия (превью, редакт, буст) доступны в один клик без дропдауна.
- Таблица вмещает на экран минимум 8 строк при стандартной высоте окна (1080p).
- При 0 профилях отображается Empty state с CTA.
- При активном бусте в строке отображается бейдж «Буст: VIP, осталось N дн.» — кнопка Буст скрыта, вместо неё в overflow «Продлить буст».

### Риски

| Риск | Митигация |
|---|---|
| Мобильная версия перестаёт совпадать по логике | `ProfileCard.tsx` использует тот же `ProfileRowActions` компонент. |
| Tooltip ломает клик по кнопке на touch-устройствах | Использовать `<Tooltip mouseEnterDelay={0.3}>`, на мобилке кнопки без тултипа. |
| Overflow-меню становится пустым при публикации + активном бусте | Явно вычислять доступные действия, если меню пустое — скрывать иконку overflow. |

## 5. Этап 5. Данные: метрики и updated_at

**Цель:** превратить страницу из «списка профилей» в дашборд с сигналом о состоянии.

### Задачи

1. Проверить, возвращает ли `/api/creators/my` поля `viewsCount` (7 дней), `applicationsCount`, `updatedAt`, `activeBoost`.
2. Если не возвращает — расширить API:
   - Бэк: [app/api/creators/route.ts](../app/api/creators/route.ts) и `/my` эндпоинт.
   - Тип: `CreatorProfile` в [lib/types/creator.ts](../lib/types/creator.ts).
   - Маппер: `mapCreatorFromApi`.
3. Добавить колонку «Обновлён» с относительным временем (`date-fns` `formatDistanceToNow` или аналог).
4. Опционально: добавить мини-колонку «Просмотры 7д» при наличии данных.

### Acceptance

- У каждой строки видно, когда профиль последний раз обновлялся.
- Если данных о просмотрах нет — колонка не отображается, а не показывает нули.

### Риски

| Риск | Митигация |
|---|---|
| Расширение API ломает другие экраны | Добавлять поля как опциональные, не менять существующие. |
| Слишком много данных в строке — снова шум | Начать с одной колонки «Обновлён», метрики добавлять только если продакт подтвердил ценность. |

## 8. Аналитика и метрики успеха

Перед началом работы ввести события:

- `manage_page_view`
- `manage_filter_change` (payload: `filter`)
- `manage_action_click` (payload: `action`, `placement: "inline" | "overflow"`)
- `manage_empty_state_cta_click`

Через 2 недели после R2 сравнить:

- CTR по «Продвинуть» (inline vs overflow в прошлом).
- Доля юзеров, использующих превью/редакт до/после.
- Среднее время до первого действия на странице.

Цель: CTR «Продвинуть» +40%, time-to-first-action −30%.
