# 📊 Сортировка креаторов в MVP

[← Назад к оглавлению](../README.md)

---

## Обзор

В MVP не используется сложный алгоритм ранжирования на сотни баллов.

Причина простая:
- на раннем этапе каталог еще маленький;
- сложный скоринг увеличивает время запуска;
- он требует множества данных, которых в MVP либо нет, либо они ненадежны;
- для быстрого запуска хватает простой и понятной логики.

---

## Логика сортировки MVP

### **1. По новизне**
Это базовая сортировка по умолчанию.

**Принцип:**
- новые профили показываются выше старых;
- это помогает новым креаторам быстрее получать первые просмотры.

### **2. По цене**
Сортировка по минимальной ставке креатора.

**Принцип:**
- более доступные креаторы выше, если бизнес ищет бюджетные варианты.

### **3. По алфавиту**
Простая вспомогательная сортировка.

---

## Ручные метки в MVP

### **Featured**
Админ может вручную выделить креатора.

Используется когда:
- профиль качественно заполнен;
- креатор уже показал себя с хорошей стороны;
- нужно помочь продвинуть сильного исполнителя.

### **Verified**
Ручная проверка профиля админом.

Используется когда:
- проверены ссылки на соцсети;
- работы выглядят реальными;
- креатор не выглядит фейком.

> В MVP `featured` и `verified` — это ручные операционные инструменты, а не автоматический скоринг.

---

## Базовые правила видимости

Профиль может быть показан в каталоге, если:
- указано имя;
- указан город;
- добавлена хотя бы 1 платформа;
- выбрана хотя бы 1 категория;
- добавлено минимум 3 работы;
- указана минимальная ставка;
- есть контакт для связи.

Если этих данных нет, профиль лучше не показывать публично.

---

## Что НЕ используется в MVP

На MVP не используем:
- скор на 380 баллов;
- сложную формулу релевантности;
- автоматическую оценку ER / Retention / охватов;
- автоштрафы и автобусты;
- AI matching;
- рекомендательные модели;
- персональные коэффициенты для каждого бизнеса.

---

## Почему это правильно для старта

Потому что сейчас важнее:
- быстро открыть каталог;
- не врать о качестве данных;
- вручную поддерживать порядок;
- собирать первые сделки;
- понять, какие фильтры реально нужны рынку.

Сложный ranking стоит добавлять только тогда, когда:
- каталог станет плотным;
- появится достаточный объем реальных данных;
- станет понятно, что ручной featured уже не справляется.

---

## Итог

На MVP каталог работает по принципу:
- **простая сортировка**;
- **ручная модерация**;
- **ручные метки featured / verified**.

Этого достаточно, чтобы быстро запуститься и не утонуть в псевдо-умной логике.
