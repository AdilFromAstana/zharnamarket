## 🎯 Что это?

**Zharnamarket PARTNER** — MVP-платформа, которая помогает бизнесу публиковать рекламные объявления, а креаторам — находить подходящие предложения и выходить на связь напрямую.

> **Главная идея:** Мы не управляем сотрудничеством. Мы связываем стороны — дальше они договариваются сами.

На текущем этапе платформа монетизируется через:
- **платную публикацию объявления** со стороны бизнеса;
- **платную публикацию профиля** со стороны креатора.

**Детали:** [`overview.md`](overview.md), [`rate-business-mvp.md`](rate-business-mvp.md)

---

## 👥 Кто пользуется платформой

> **Важно:** Нет ролей. Нет переключения режимов. Один аккаунт = один человек. Он может создавать задания И иметь профиль креатора — оба действия доступны всегда.

**Подробнее о пересмотре ролей:** [`redesign-roles.md`](redesign-roles.md)

### 1. **Хочет найти креатора**
Любой пользователь может:
- создать задание (объявление) с описанием что нужно;
- оплатить публикацию на 7 дней;
- получать входящие обращения напрямую;
- самостоятельно выбирать с кем работать.

**Детали:** [`roles/advertiser.md`](roles/advertiser.md)

---

### 2. **Хочет предложить свои услуги**
Любой пользователь может:
- заполнить профиль с примерами работ;
- просматривать задания и писать напрямую;
- опционально опубликоваться в каталоге (платно);
- получать входящие от тех, кто нашёл профиль.

**Детали:** [`roles/creator.md`](roles/creator.md)

## 🔄 Основные процессы

### **Для бизнеса:**

```text
Регистрация → Email верификация (6-значный код) → Создание объявления →
Оплата публикации → Объявление в ленте (7 дней) → Входящие обращения →
Общение в Telegram/WhatsApp
```

Бизнес сам управляет объявлением: пауза, возобновление, архив, удаление. Автоистечение через 7 дней.

### **Flow регистрации (реализовано):**

```text
1. Пользователь заполняет форму: имя, телефон, email, пароль
2. POST /api/auth/register → user создан с emailVerified: false
3. Генерация 6-значного кода → SHA-256 хеш в БД → код на email
4. Redirect на /auth/verify-email?email=xxx
5. Пользователь вводит код (6 цифр, OTP-поля, автосубмит)
6. POST /api/auth/verify-email → emailVerified: true → JWT токены → вход

Альтернатива: Google OAuth → emailVerified: true автоматически → сразу вход
```

### **Детальный flow создания объявления (5 шагов):**

```text
Шаг 0: Онбординг (только при первом создании)
  → Имя, Telegram
  → Сохраняется в localStorage

Шаг 1: Основная информация
  → Название (10+), платформа, город, категория, описание (50+), фото (до 5)
  → Автосохранение в localStorage(new_ad_draft) при каждом изменении

Шаг 2: Бюджет и контакты
  → Тип бюджета (фикс/за просмотры/% от продаж/по договорённости)
  → Суммы, Telegram (обяз.), WhatsApp (опц.)
  → Обновляется в localStorage(new_ad_draft)

Шаг 3: Предпросмотр
  → Карточка объявления как увидят креаторы
  → Кнопка «К оплате»

Шаг 4: Оплата публикации
  → Цена: 990 ₸ / 7 дней
  → Выбор способа: Kaspi / Halyk / карта
  → Поле промокода (опционально, валидация в реальном времени)
  → Если промокод 100% → бесплатная публикация
  → Если нет → redirect на платёжный провайдер (mock: мгновенно)
  → Webhook → статус active, publishedAt, expiresAt (+7 дней)
  → Буст — докупается отдельно из /ads/manage после публикации

Жизненный цикл объявления:
  draft → [оплата] → active (7 дней) → expired
                        ↕ pause/resume
                        → archived
           любой → deleted (soft)
```

### **Flow оплаты профиля креатора (реализовано):**

```text
1. Пользователь создаёт профиль → isPublished: false (черновик)
2. На /creators/manage нажимает «Опубликовать (990 ₸)»
3. Модалка: выбор способа оплаты + промокод
4. POST /api/payments/creators/[id]/publish
5. Mock: redirect → webhook → isPublished: true
6. Redirect на /creators/manage?payment=success
7. Toast «Профиль опубликован!»
```

### **Для креатора:**

**Вариант А:** Найти объявление
```text
Регистрация → Создание профиля → Просмотр объявлений → Нажатие «Связаться» →
Переход во внешний мессенджер → Прямое общение с бизнесом
```

**Вариант Б:** Получение приглашения
```text
Регистрация → Создание портфолио → Оплата публикации профиля → Появление в каталоге →
Получение входящего от бизнеса в мессенджере
```

## 💰 Монетизация MVP

### **Основная модель**
- **Платная публикация объявления**

### **Актуальные тарифы**

| Опция | Цена | Срок | Особенности |
|-------|------|------|-------------|
| **Публикация** | 990 ₸ | 7 дней | Обязательная базовая публикация, тест платформы |
| **Буст** | доп. цена | 7 дней | Опционально из ЛК: поднятие в ленте, бейдж, выделение цветом |

> Логика аналогична OLX: сначала базовая публикация (990 ₸), потом опционально буст. Низкий барьер входа снижает риск первой оплаты для нового бизнеса.

> Бизнес платит за размещение и доступ к аудитории, а не за гарантированные сделки. Мы — платная доска объявлений, а не арбитр сотрудничества.

**Детали:** [`monetization/models.md`](monetization/models.md), [`rate-business-mvp.md`](rate-business-mvp.md)

- категории;

Минимально включает:

- описание;

- бюджет или ставку;

- требования;
- контакт для связи;
- статус публикации.
