## Концепция

**ViralAds PARTNER** — MVP-платформа для быстрого соединения бизнеса и креаторов в Казахстане.

> **Ключевая идея:** Мы не управляем сделками. Мы — платная доска объявлений с прямым контактом.

Платформа решает две простые задачи:
1. **Для бизнеса:** опубликовать рекламное объявление и получить прямые обращения от заинтересованных креаторов
2. **Для креаторов:** найти подходящее предложение и быстро выйти на контакт с работодателем

Продукт не строится вокруг автоматического performance-трекинга или управления сотрудничеством. Он строится вокруг **платного размещения объявлений и прямого контакта**.

## Ключевые особенности MVP

### 🎬 **Мультиплатформенность**
Поддержка короткого видео:
- TikTok
- Instagram Reels
- YouTube Shorts

### 🌍 **Локальный фокус**
Старт с городами Казахстана:
- Алматы
- Астана
- Шымкент
- Караганда
- Актау
- Павлодар

### 🔄 **Двусторонний механизм**

**Модель А:** Бизнес публикует объявление → Креаторы находят его и пишут напрямую
**Модель Б:** Креаторы публикуют портфолио → Бизнес находит их и пишет напрямую

В обоих случаях контакт — прямой. Платформа не посредник в переговорах.

### 👤 **Единый аккаунт — любые действия**
- один номер телефона + один email = один аккаунт;
- можно разместить задание И создать профиль — никакого выбора «кто ты»;
- нет ролей, нет переключения режимов — просто нажми нужное действие.

**Подробнее:** [`redesign-roles.md`](redesign-roles.md)

### 🛡️ **Базовая защита MVP**
- модерация объявлений;
- ручная блокировка мусорных пользователей;
- ручная проверка сильных креаторов;
- запрет self-dealing сценариев.

## Целевая аудитория

### **Креаторы:**
- активные авторы в TikTok / Reels / Shorts;
- хотят монетизировать контент;
- готовы работать с рекламными заказами;
- находятся в Казахстане.

### **Бизнес:**
- малый и средний бизнес;
- локальные бренды и сервисы;
- заведения, магазины, онлайн-услуги;
- те, кому нужен быстрый доступ к креаторам.

---

## Ценностное предложение MVP

### Для бизнеса:
- простая платная публикация объявления на 7 дней;
- доступ к аудитории реальных креаторов;
- входящий поток прямых обращений;
- самостоятельное управление объявлением.

### Для креаторов:
- простое портфолио;
- живая лента рекламных предложений;
- прямой контакт с бизнесом без посредников;
- быстрый старт без бюрократии.

---

## Основные метрики MVP

### Для платформы:
- количество опубликованных объявлений (платных);
- выручка от публикаций;
- количество нажатий «Связаться»;
- количество активных креаторов;
- количество активных бизнесов.

### Что не считаем в MVP как системную истину:
- просмотры видео;
- клики;
- продажи;
- ROI;
- CPL / CPA-атрибуцию внутри платформы.

## Технологический стек

### Full-Stack (Next.js monolith):
- **Next.js 16** (App Router) — фронтенд и бэкенд в одном проекте
- **TypeScript** — строгая типизация
- **Prisma ORM 7** + **PostgreSQL** — база данных
- **Ant Design 5** + **Tailwind CSS 4** — UI
- **JWT (jose, HS256)** — аутентификация (access 15мин + refresh 30дн)
- **Google OAuth 2.0** — вход через Google
- **Nodemailer (SMTP)** — email: верификация (6-значный код), сброс пароля
- **Vitest 4** — 137 интеграционных тестов с реальной БД
- **proxy.ts** — серверная защита маршрутов (Next.js 16 Edge)

### Оплата:
- Абстракция `PaymentProvider` — единый интерфейс для любого провайдера
- Mock-провайдер для разработки (мгновенная оплата)
- Готово к подключению Kaspi Pay / Halyk Bank (1 класс + 1 env переменная)

## Бизнес-модель MVP

### Основная монетизация:
1. **Платная публикация объявлений** — основная
2. **Платная публикация профиля креатора** — дополнительная
3. **Boost / Featured размещение** — апселл

> MVP не монетизируется через performance billing и не участвует в расчетах между сторонами.

---

## Как выглядит жизненный цикл объявления

```text
Создание → Оплата → Активация → Входящие обращения → Истечение срока
                                      ↓
                              Пауза (опционально)
                                      ↓
                         Архивирование / Удаление вручную
```

Объявление живет срок, за который заплатил бизнес. Дальше — его решение.

## Потенциал роста

### Фаза 1 (MVP) — РЕАЛИЗОВАНО:
- регистрация + email верификация (6-значный код) + Google OAuth;
- публикация объявлений (5-шаговый wizard, автосохранение, 7 статусов);
- каталог креаторов (фильтры, публикация профилей);
- оплата публикации объявлений и профилей (mock-провайдер, промокоды);
- буст объявлений (3 типа);
- прямой контакт через кнопку «Связаться» + tracking;
- отзывы и рейтинг;
- подача жалоб;
- админ-панель (дашборд, промокоды);
- серверная защита маршрутов;
- 137 интеграционных тестов.

### Фаза 2 (следующая):
- реальный платёжный провайдер (Kaspi/Halyk);
- админ: управление пользователями, модерация, обработка жалоб;
- текстовый поиск по каталогам;
- верификация креаторов;
- облачное хранилище файлов.

### Фаза 3:
- расширение географии;
- мессенджер внутри платформы;
- Telegram Login / SMS верификация;
- аналитика с графиками.

## Риски MVP

| Риск | Вероятность | Митигация |
|------|-------------|-----------|
| Мусорные объявления | Высокая | Ручная модерация |
| Мало входящих обращений для бизнеса | Средняя | Улучшение формы объявления и онбординга креаторов |
| Плохое качество профилей | Средняя | Минимальные требования к портфолио |
| Низкое доверие | Средняя | Ручная верификация и featured/verified метки |
| Конкуренция | Средняя | Локальный фокус и быстрый запуск |
