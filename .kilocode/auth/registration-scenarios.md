## Общий подход

**Один аккаунт = один пользователь.** Нет ролей. Нет «активного режима». Нет «переключи на бизнес».

Пользователь может:
- разместить задание (ищет создателя контента)
- создать профиль (сам является создателем контента)
- делать оба действия — без каких-либо ограничений

Регистрация **минимальная**: данные → аккаунт создан → старт.

## Сценарий 1: Регистрация (любой пользователь)

### **Шаг 1: Базовые данные**

```
┌────────────────────────────────┐
│      viral-wall                │
│                                │
│   Создание аккаунта            │
│                                │
│   Имя: [____________]          │
│   Телефон: +7 [________]       │
│   Email: [____________]        │
│   Пароль: [____________]       │
│                                │
│   [ ] Согласен с условиями     │
│                                │
│   [Создать аккаунт]            │
│                                │
│   Уже есть аккаунт? [Войти]    │
└────────────────────────────────┘
```

**Что происходит:**
- Создаётся аккаунт User
- Отправляется SMS-код для верификации (или пропускается в MVP)
- Пользователь попадает на главную страницу

**Нет шага «Выбор роли».** Страница `/auth/role` — удалена.

## Сценарий 2: Пользователь хочет разместить задание

*Например: владелец кофейни хочет найти TikTok-блогера*

### **Точка входа**
- Кнопка «Разместить задание» на главной
- Кнопка «+ Новое задание» в кабинете
- Ссылка «Разместить объявление →» в шапке или баннере

### **Первое размещение (нет бизнес-профиля)**

```
Шаг 0 (только при первом размещении):
┌────────────────────────────────┐
│   Быстрая настройка            │
│                                │
│   Как к вам обращаться?        │
│   [________________________]   │
│   (имя, компания или nickname) │
│                                │
│   Telegram для входящих:       │
│   [@_____________]             │
│                                │
│   [Продолжить →]               │
└────────────────────────────────┘

Шаг 1: Основная информация
  → Название задания
  → Платформа (TikTok / Reels / Shorts)
  → Город
  → Категория
  → Описание

Шаг 2: Детали и контакты
  → Бюджет/ставка
  → Требования
  → Дедлайн
  → Telegram/WhatsApp для связи

Шаг 3: Предпросмотр

Шаг 4: Оплата (990 ₸ / 7 дней)
  → Kaspi / Halyk / карта
  → После оплаты: задание активно ✅
```

### **Повторное размещение (есть бизнес-профиль)**

```
Шаг 1 → Шаг 2 → Шаг 3 → Шаг 4
(Шаг 0 пропускается автоматически)
```

## Сценарий 3: Пользователь хочет создать профиль креатора

*Например: TikTok-блогер хочет попасть в каталог*

### **Точка входа**
- Кнопка «Создать профиль» в `/creators`
- Кнопка «Мой профиль» в кабинете
- Если профиля нет → предлагается создать

### **Онбординг профиля**

```
Шаг 1: О себе
  → Город
  → Платформы (TikTok, Reels, Shorts)
  → Категории контента
  → Telegram для связи

Шаг 2: Портфолио
  → Минимум 3 примера работ
  → Ссылки или загрузка

Шаг 3: Ставка и доступность
  → Минимальная ставка
  → Статус: Свободен / Частично / Занят

Шаг 4: Публикация (опционально)
  → Профиль можно НЕ публиковать в каталоге
  → Публикация в каталоге = платная (опционально)
  → [Опубликовать за 990 ₸] или [Сохранить без публикации]
```

## Сценарий 4: Пользователь делает оба (владелец bakery + сам снимает)

*Пример: кофейня, которая сама снимает контент и ищет других блогеров*

Нет никакого специального сценария. Пользователь просто:
1. Создаёт задание (находит других креаторов)
2. Создаёт профиль (предлагает свои услуги другим)

В кабинете оба блока видны и управляются отдельно.


## Работа с аккаунтом

### **Если нужно добавить бизнес-контакты позже**

Просто создать первое задание — шаг 0 спросит контакт.

### **Если нужно создать профиль позже**

Зайти в `/cabinet` → нажать «Создать профиль» → онбординг.

### **Защита от конфликтов**

❌ Нажать «Связаться» на своё задание — система блокирует с пояснением.

## Техническая реализация

### **База данных (упрощённая модель):**

```sql
CREATE TABLE users (
  id UUID PRIMARY KEY,
  email VARCHAR UNIQUE,
  phone VARCHAR UNIQUE,
  password_hash VARCHAR,
  name VARCHAR,
  avatar VARCHAR,
  -- НЕТ: active_role, is_creator, is_advertiser
  created_at TIMESTAMP,
  updated_at TIMESTAMP
);

CREATE TABLE business_contacts (
  -- Контактная информация для заданий (создаётся при первом задании)
  user_id UUID PRIMARY KEY REFERENCES users(id),
  display_name VARCHAR,  -- «Кофейня Алишера» или «Алишер»
  telegram VARCHAR,
  whatsapp VARCHAR,
  created_at TIMESTAMP
);

CREATE TABLE creator_profiles (
  user_id UUID PRIMARY KEY REFERENCES users(id),
  city VARCHAR,
  platforms JSONB,
  categories JSONB,
  min_rate DECIMAL,
  telegram VARCHAR,
  whatsapp VARCHAR,
  availability VARCHAR,  -- 'free' | 'partial' | 'busy'
  is_published BOOLEAN DEFAULT false,
  verified BOOLEAN DEFAULT false,
  featured BOOLEAN DEFAULT false
);

CREATE TABLE ads (
  id UUID PRIMARY KEY,
  owner_id UUID REFERENCES users(id),
  title VARCHAR,
  description TEXT,
  platform VARCHAR,
  city VARCHAR,
  category VARCHAR,
  budget VARCHAR,
  deadline VARCHAR,
  requirements TEXT,
  contact_telegram VARCHAR,
  contact_whatsapp VARCHAR,
  tariff ENUM('basic', 'boost', 'featured'),
  status ENUM('draft', 'pending_payment', 'active', 'paused', 'expired', 'archived', 'deleted'),
  published_at TIMESTAMP,
  expires_at TIMESTAMP,
  view_count INTEGER DEFAULT 0,
  contact_click_count INTEGER DEFAULT 0,
  featured BOOLEAN DEFAULT false,
  created_at TIMESTAMP,
  updated_at TIMESTAMP
);
```

## Итого: Что делает пользователь

### **При регистрации:**
1. ✅ Ввести имя, телефон, email, пароль
2. ✅ Начать работу (нет шага «Выбор роли»)

### **При первом размещении задания:**
1. ✅ Нажать «Создать задание»
2. ✅ [Первый раз] Ввести имя/контакт (30 секунд)
3. ✅ Заполнить форму задания
4. ✅ Оплатить публикацию

### **При создании профиля:**
1. ✅ Нажать «Создать профиль»
2. ✅ Заполнить онбординг
3. ✅ [Опционально] Оплатить публикацию в каталоге

### **Ключевое:**
Один и тот же пользователь может и создавать задания, и иметь профиль. Никакого «выбери кем быть».
